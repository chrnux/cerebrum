Tegnsett, spesial-tegn, quoting
-------------------------------

Klienten forstår både enkle og doble fnutter, men særbehandler ikke
backslash på noen måte i slike kontekster.

Vil angi: foo"bar'xyzzy med \backslash
Quotes som: foo\"bar\'xyzzy\ med\ \\backslash
eller: 'foo"bar\'xyzzy med \\backslash'
osv.


Default verdier/Optional attributter
------------------------------------

Med spesifikasjonen menes her den tekstlige beskrivelsen av kommandoer
i dette dokumentet.

Argumenter som muligens har default-verdi angis i spesifikasjonen med
"=" på slutten.  Argumenter som er valgfrie, angis inne i [].

  * Hvis man har oppgitt så få argumenter at noen gjenværende
    argumenter har default verdi (evt er vanlige parametere), vil man
    kastes inn i prompt modus.  Klienten spør da serveren både hva
    promptet skal være, og etter dens default verdi (hvis det viser
    seg hensiktsmessig, kan noen av disse verdiene encodes i Command
    objektet slik at klienten ikke trenger spørre severen).
  * Dersom man tidligere på komandolinjen har oppgitt en tuple, gir
    ikke serveren noen default verdi.
  * Klienten husker default verdien, og sender denne til serveren.
  * Når man i klienten taster <enter> for en parameter som har default
    verdi, settes default verdi inn.
  * Når man i klienten taster <enter> for en parameter som er
    optional, sendes den tomme strengen. (TBD: dette gjør det umulig å
    sende "null".  Er det OK?  Kan løses ved å angi noe i Parameter
    objektet)
  * Dersom det bare er optional paremetere igjen på komandolinjen,
    havner man ikke i prompt modus.
  * Har man først havnet i prompt modus vil man promptes for alle
    resterende parametere.

I denne versjonen implementerer vi ikke tab-completion.  Det kommer
trolig på et senere tidspunkt, og vil sansynligvis være en enkel
utvidelse av funksjonaliteten for å hente default-verdi.

Person - Create
---------------

 I tillegg til de personer som systemet får via autorative systemer må
 en kunne legge inn helt egne personentyer.


# Gender will default to 'unknown' for persons with no fnr, if not
# given here.

bofh> person bcreate <display_name> <birth_date (yyyy-mm-dd)> [gender (m|f)]
Created 'display_name' as person with birthdate yyyy-mm-dd.

 Warning:
   - Finnes person med navn og fødseldato fra før.  Skal kun bygge
     efter en bekreftelse fra bruker av bofh.


bofh> person fcreate <display_name> <NoSSN (19027732244)>
Created 'display_name' as person whith fnr 'fnr'.

 Feil:
   - Det finnes alt en person med dette fødselnummeret


bofh> person create <display_name> <id_type> <id> 
Created 'display_name' as person with id 'export_id' (name).

 Feil:
   - Det finnes alt en person med denne ID av denne typen.
